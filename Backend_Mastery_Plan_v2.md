# Backend Mastery & AWS SAA Roadmap (v2)

## Core Philosophy (unchanged)
1. **Build First, Theory Second** — you already know how to build. Fill gaps, don't relearn basics.
2. **One Primary Focus:** Java Fundamentals & DSA.
3. **One Secondary Focus:** AWS SAA — passive, downtime only, never competes with coding time.
4. Spring Boot / Security internals is tracked separately (ongoing Claude sessions using MPPS) — not duplicated here.

---

## Right Now (Active Focus)

**1. Exercism — Java Track**
Status: Basics, Strings, Booleans, If-Else, Numbers ✅ done.
Next: Nullability → Classes → Ternary Operators → Arrays.

Rule: don't grind every locked concept just because it's there. If MPPS work already proves you're solid on something (Lists, Maps, Interfaces, Exceptions — you use these daily), skip the concept exercise and move on. Only stop and drill where you actually hit friction or interview gaps.

**2. Start DSA (Fiset playlist) as soon as core Exercism concepts are solid — don't wait for 100%**
Run this in parallel with the tail end of Exercism, not strictly after. Watching *why* HashMaps/Arrays/Trees behave the way they do while still writing idiomatic Java reinforces both at once.

**3. LeetCode (NeetCode 150 patterns)**

## Daily / Weekly Routine

- **Work Hours:** Build features, apply what you're learning in real code.
- **Evening (1–2 hrs):** Active coding — Exercism → DSA playlist → (later) LeetCode.
- **Downtime (30 min):** AWS SAA video, passive only.
- **Weekend:** Update `backend-interview-mastery` repo.

## Guardrails (why this version changed)
- Two tracks are running (this one + Spring Boot/Security internals elsewhere). That's fine as long as they're sequenced and bounded — not the "learn everything at once" pattern that caused the original overwhelm.
- AWS SAA is valuable because the bank pays for it, not because it builds the kind of depth you're chasing. Keep it in its lane.
- Career direction: backend-first applications. Frontend stays a genuine asset on the resume, not something you split prep time defending in interviews.
